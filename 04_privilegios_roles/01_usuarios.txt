La B.D. debe proveer de mecanismos de seguridad para garantizar quién entra (autenticación) y para decirle qué 

puede hacer una vez que ha sido autenticado (autorización). Independientemente de lo que podamos llegar a tener 

como herramientas de seguridad informática, esto es propio de la B.D.


	* Autenticación: identifica al usuario. 
	
	Ya sea a través de un usuario y password, un certificado, usuarios de S.O. etc.


	* Autorización: qué puede hacer el usuario a través privilegios.


En todos los S.G.B.D. se hacen dos pasos, se crea primero el usuario y luego se le asignan privilegios.




Para llevar una correcta gestión de autenticación y autorización hay una herramienta llamada matriz de 

autorización definida en el ISO 27000, norma para la seguridad de la información, es decir, para garantizar la 

integridad de los datos. La matriz de autorización es una tarjeta con todos los roles y accesos posibles:


	* Lista de todos los roles / perfiles


	* Lista de todos los aplicativos de la organización
	
	En nuestro caso, de todas las B.D. que puede haber en la organización, incluidas diferentes tecnologías.

	
	* Modelo de política de accesos

	Qué perfil tiene o no acceso a qué.


	* Conocer todo el panorama


	* Borrar y corregir


	* Realizar los puntos anteriores periódicamente




Para el control de acceso se requiere:


* Política de seguridad

	Mandato general: qué servicios se van a cuidar, con qué se van a cuidar, con qué tecnología; de aquí surge 

	qué se puede y el presupuesto. Incluye ciberseguridad, seguridad informática, seguridad y protección de 

	datos.

	
* Modelo de seguridad

	ISO 27000, IEEE, etc.


* Mecanismo de seguridad

	Algoritmo o funcionalidad que vamos a definir. Hay que tener un inventario de dónde surge el mecanismo, 

	bajo qué modelo lo estoy implementado y de qué política lo extraigo. Todo debe tener una cuestión de 

	rastreo, de donde surge y por qué está ahí.




Privilegios: lo que puede hacer un usuario con los objetos de la B.D. 

Crear, listar, insertar, ejecutar, mostrar, modificar, etc.


* Nivel administrativo

	Todo lo que tiene que ver con el server. Es a nivel global, estas políticas o privilegios son globales para 

	todo el servidor. Sólo un usuario debe contar con estos privilegios y es el D.B.A.


* Nivel B.D.

	B.D. como objeto general, es decir, qué puedo hacer con ella como crear, listar, eliminar, cambiar, etc. 

	Aquí puede ser a nivel global o particular; puede haber más de un usuario con privilegios para B.D.


* Nivel Objetos

	Modificaciones sobre tablas, vistas, rutinas, triggers, funciones, etc. Las asignaciones a los usuarios 

	son de forma granular.
